21 // BACKGROUND:
22 // When I first received an ultrasonic sensor I was not happy with how poorly
23 // it worked. Quickly I realized the problem wasn't the sensor, it was the
24 // available ping and ultrasonic libraries causing the problem. The NewPing
25 // library totally fixes these problems, adds many new features, and breaths
26 // new life into these very affordable distance sensors.
27 //
28 // FEATURES:
29 // * Works with many different ultrasonic sensors: SR04, SRF05, SRF06, DYP-ME007, URM37 & Parallax PING)))â„¢.
30 // * Compatible with the entire Arduino line-up (and clones), Teensy family (including $19.80 96Mhz 32 bit Teensy 3.2) and non-AVR microcontrollers.
31 // * Interface with all but the SRF06 sensor using only one Arduino pin.
32 // * Doesn't lag for a full second if no ping/echo is received.
33 // * Ping sensors consistently and reliably at up to 30 times per second.
34 // * Timer interrupt method for event-driven sketches.
35 // * Built-in digital filter method ping_median() for easy error correction.
36 // * Uses port registers for a faster pin interface and smaller code size.
37 // * Allows you to set a maximum distance where pings beyond that distance are read as no ping "clear".
38 // * Ease of using multiple sensors (example sketch with 15 sensors).
39 // * More accurate distance calculation (cm, inches & uS).
40 // * Doesn't use pulseIn, which is slow and gives incorrect results with some ultrasonic sensor models.
41 // * Actively developed with features being added and bugs/issues addressed.
42 //
43 // CONSTRUCTOR:
44 // NewPing sonar(trigger_pin, echo_pin [, max_cm_distance])
45 // trigger_pin & echo_pin - Arduino pins connected to sensor trigger and echo.
46 // NOTE: To use the same Arduino pin for trigger and echo, specify the same pin for both values.
47 // max_cm_distance - [Optional] Maximum distance you wish to sense. Default=500cm.
48 //
49 // METHODS:
50 // sonar.ping([max_cm_distance]) - Send a ping and get the echo time (in microseconds) as a result. [max_cm_distance] allows you to optionally set a new max distance.
51 // sonar.ping_in([max_cm_distance]) - Send a ping and get the distance in whole inches. [max_cm_distance] allows you to optionally set a new max distance.
52 // sonar.ping_cm([max_cm_distance]) - Send a ping and get the distance in whole centimeters. [max_cm_distance] allows you to optionally set a new max distance.
53 // sonar.ping_median(iterations [, max_cm_distance]) - Do multiple pings (default=5), discard out of range pings and return median in microseconds. [max_cm_distance] allows you to optionally set a new max distance.
54 // NewPing::convert_in(echoTime) - Convert echoTime from microseconds to inches (rounds to nearest inch).
55 // NewPing::convert_cm(echoTime) - Convert echoTime from microseconds to centimeters (rounds to nearest cm).
56 // sonar.ping_timer(function [, max_cm_distance]) - Send a ping and call function to test if ping is complete. [max_cm_distance] allows you to optionally set a new max distance.
57 // sonar.check_timer() - Check if ping has returned within the set distance limit.
58 // NewPing::timer_us(frequency, function) - Call function every frequency microseconds.
59 // NewPing::timer_ms(frequency, function) - Call function every frequency milliseconds.
60 // NewPing::timer_stop() - Stop the timer.
61 //

63 // 12/09/2017 v1.9.0 - Added support for the ARM-based Particle devices. If
64 // other ARM-based microcontrollers adopt a similar timer method that the
65 // Particle and Teensy 3.x share, support for other ARM-based microcontrollers
66 // could be possible. Attempt to add to Arduino library manager. License
67 // changed.
68 // 07/30/2016 v1.8 - Added support for non-AVR microcontrollers. For non-AVR
69 // microcontrollers, advanced ping_timer() timer methods are disabled due to
70 // inconsistencies or no support at all between platforms. However, standard
71 // ping methods are all supported. Added new optional variable to ping(),
72 // ping_in(), ping_cm(), ping_median(), and ping_timer() methods which allows
73 // you to set a new maximum distance for each ping. Added support for the
74 // ATmega16, ATmega32 and ATmega8535 microcontrollers. Changed convert_cm()
75 // and convert_in() methods to static members. You can now call them without
76 // an object. For example: cm = NewPing::convert_cm(distance);
77 //
78 // 09/29/2015 v1.7 - Removed support for the Arduino Due and Zero because
79 // they're both 3.3 volt boards and are not 5 volt tolerant while the HC-SR04
80 // is a 5 volt sensor. Also, the Due and Zero don't support pin manipulation
81 // compatibility via port registers which can be done (see the Teensy 3.2).
82 //
83 // 06/17/2014 v1.6 - Corrected delay between pings when using ping_median()
84 // method. Added support for the URM37 sensor (must change URM37_ENABLED from
85 // false to true). Added support for Arduino microcontrollers like the $20
86 // 32 bit ARM Cortex-M4 based Teensy 3.2. Added automatic support for the
87 // Atmel ATtiny family of microcontrollers. Added timer support for the
88 // ATmega8 microcontroller. Rounding disabled by default, reduces compiled
89 // code size (can be turned on with ROUNDING_ENABLED switch). Added
90 // TIMER_ENABLED switch to get around compile-time "__vector_7" errors when
91 // using the Tone library, or you can use the toneAC, NewTone or
92 // TimerFreeTone libraries: https://bitbucket.org/teckel12/arduino-toneac/
93 // Other speed and compiled size optimizations.
94 //
95 // 08/15/2012 v1.5 - Added ping_median() method which does a user specified
96 // number of pings (default=5) and returns the median ping in microseconds
97 // (out of range pings ignored). This is a very effective digital filter.
98 // Optimized for smaller compiled size (even smaller than sketches that
99 // don't use a library).
100 //
101 // 07/14/2012 v1.4 - Added support for the Parallax PING)))ï¿½ sensor. Interface
102 // with all but the SRF06 sensor using only one Arduino pin. You can also
103 // interface with the SRF06 using one pin if you install a 0.1uf capacitor
104 // on the trigger and echo pins of the sensor then tie the trigger pin to
105 // the Arduino pin (doesn't work with Teensy). To use the same Arduino pin
106 // for trigger and echo, specify the same pin for both values. Various bug
107 // fixes.
108 //
109 // 06/08/2012 v1.3 - Big feature addition, event-driven ping! Uses Timer2
110 // interrupt, so be mindful of PWM or timing conflicts messing with Timer2
111 // may cause (namely PWM on pins 3 & 11 on Arduino, PWM on pins 9 and 10 on
112 // Mega, and Tone library). Simple to use timer interrupt functions you can
113 // use in your sketches totally unrelated to ultrasonic sensors (don't use if
114 // you're also using NewPing's ping_timer because both use Timer2 interrupts).
115 // Loop counting ping method deleted in favor of timing ping method after
116 // inconsistent results kept surfacing with the loop timing ping method.
117 // Conversion to cm and inches now rounds to the nearest cm or inch. Code
118 // optimized to save program space and fixed a couple minor bugs here and
119 // there. Many new comments added as well as line spacing to group code
120 // sections for better source readability.
121 //
122 // 05/25/2012 v1.2 - Lots of code clean-up thanks to Arduino Forum members.
123 // Rebuilt the ping timing code from scratch, ditched the pulseIn code as it
124 // doesn't give correct results (at least with ping sensors). The NewPing
125 // library is now VERY accurate and the code was simplified as a bonus.
126 // Smaller and faster code as well. Fixed some issues with very close ping
127 // results when converting to inches. All functions now return 0 only when
128 // there's no ping echo (out of range) and a positive value for a successful
129 // ping. This can effectively be used to detect if something is out of range
130 // or in-range and at what distance. Now compatible with Arduino 0023.
131 //
132 // 05/16/2012 v1.1 - Changed all I/O functions to use low-level port registers
133 // for ultra-fast and lean code (saves from 174 to 394 bytes). Tested on both
134 // the Arduino Uno and Teensy 2.0 but should work on all Arduino-based
135 // platforms because it calls standard functions to retrieve port registers
136 // and bit masks. Also made a couple minor fixes to defines.
